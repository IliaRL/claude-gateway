# Plan 02-01: Security Audit — SEC-01/02/03

## Objective

Run a three-part security audit:
- **SEC-01:** Verify no hardcoded API keys or OAuth tokens in source files
- **SEC-02:** Verify no command injection paths in shell exec calls
- **SEC-03:** Audit credential sync scripts for secure patterns; fix the stale CONFIGS_DIR path bug in sync-kiro-credentials.py

## Context

Research found:
- SEC-01: All credentials flow through `provider_pools.json` (runtime-only) or `Credentials/` files — no hardcoded literal tokens expected
- SEC-02: exec/fork callers are `update-api.js` (git ops with system-controlled args) and `master.js` (fork with config-driven args) — no user HTTP input reaches exec
- SEC-03: `sync-credentials.js` already has path traversal guard; `sync-kiro-credentials.py` has a stale path (`Tier1-AIClient2API` → should be `AIClient2API`)

<read_first>

Run the following greps from the MASTER-C/ directory:

```bash
grep -rn "sk-[a-zA-Z0-9]\{20,\}" AIClient2API/src/ AIClient2API/configs/config.json
```

Do NOT grep configs/kiro/ or Credentials/ — those are credential storage, not source code.

For any match: determine if it's a hardcoded literal token or a template/placeholder.
- Placeholders like `"Bearer $AICLIENT_TOKEN"` or `"sk-{api_key}"` are NOT violations.
- Literal tokens (long random strings) ARE violations — document and flag for manual rotation.
</action>
