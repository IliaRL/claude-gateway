<objective>
Fix the empty-tool-name streaming bug in OpenAIConverter.toClaudeStreamChunk so that
content_block_start for a tool_use block is emitted exactly once per tool index, and
that any previously open block (text OR parallel tool) is properly closed before
opening a new tool block.

Purpose: When an OpenAI provider streams tool calls, the first chunk carries a non-empty
function.name and subsequent chunks carry empty function.name with more argument JSON.
The current code at line 581 only emits content_block_start when toolCall.function?.name
is truthy — which is correct. However, it does NOT currently check whether
toolIndexMap.has(openaiIdx) before opening a new block. If a provider sends the name
in multiple chunks, a duplicate block_start fires.

There is a second related bug: the original fix proposal (adding `!toolIndexMap.has` to
the outer guard but keeping the inner `if (currentBlockType !== 'tool_use')` branch)
fails for PARALLEL tool calls. When tool index 0 is open (currentBlockType='tool_use')
and tool index 1 starts, the outer guard passes (new openaiIdx) but the inner
`currentBlockType !== 'tool_use'` check is false — so no content_block_stop is emitted
for tool 0 before tool 1 opens. This violates the Anthropic SSE protocol.

The correct fix consolidates both close branches into a single `if (state.blockStarted)`
check, which correctly handles all transitions:
  1. text block open → new tool: blockStarted=true, currentBlockType='text' → close fires
  2. tool block open → new parallel tool: blockStarted=true, new openaiIdx → close fires
  3. no open block → first tool: blockStarted=false → close skipped
  4. same tool repeated (same openaiIdx): outer guard blocks entire body → nothing

Output: OpenAIConverter.js with the tool block guard hardened at lines ~581-606.
</objective>

CURRENT code around lines 578-616 (the tool call section inside toClaudeStreamChunk):
```javascript
            if (toolCall.function?.name) {
                // New tool call starting: close any open text/thinking block first.
                if (state.blockStarted && state.currentBlockType !== 'tool_use') {
                    events.push({ type: "content_block_stop", index: state.blockIndex });
                    state.blockIndex++;
                    state.blockStarted = false;
                } else if (state.blockStarted && state.toolIndexMap.has(openaiIdx)) {
                    // Parallel tool: close previous tool block before opening next.
                    events.push({ type: "content_block_stop", index: state.blockIndex });
                    state.blockIndex++;
                    state.blockStarted = false;
                }
                state.toolIndexMap.set(openaiIdx, state.blockIndex);
                state.blockStarted = true;
                state.currentBlockType = 'tool_use';
                events.push({
                    type: "content_block_start",
                    index: state.blockIndex,
                    content_block: {
                        type: "tool_use",
                        id: toolCall.id || `tool_${uuidv4()}`,
                        name: toolCall.function.name,
                        input: {}
                    }
                });
            }

            if (toolCall.function?.arguments) {
                const blockIdx = state.toolIndexMap.get(openaiIdx) ?? state.blockIndex;
                events.push({
                    type: "content_block_delta",
                    index: blockIdx,
                    delta: { type: "input_json_delta", partial_json: toolCall.function.arguments }
                });
            }
```

THE BUG (two issues):

1. The outer `if (toolCall.function?.name)` has no toolIndexMap guard, so if a provider
   sends non-empty name on every chunk for the same tool index, content_block_start fires
   multiple times for the same tool.

2. The inner split between `if (currentBlockType !== 'tool_use')` and `else if (toolIndexMap.has(openaiIdx))`
   breaks parallel tools. When openaiIdx=1 starts while openaiIdx=0 is open:
   - Outer `if (toolCall.function?.name)` passes (name is truthy)
   - But toolIndexMap.has(openaiIdx=1) is FALSE — so the else-if never fires
   - currentBlockType is 'tool_use' — so the first if never fires
   - Result: openaiIdx=0 block is NEVER closed before openaiIdx=1 opens → malformed stream

THE FIX: Add `!state.toolIndexMap.has(openaiIdx)` to the outer condition AND replace
both inner close branches with a single `if (state.blockStarted)` check:

```javascript
            if (toolCall.function?.name && !state.toolIndexMap.has(openaiIdx)) {
                // Close whatever block was open previously (text block OR previous parallel tool).
                // This unified check handles all transitions:
                //   text→tool, thinking→tool, tool[n]→tool[n+1] (parallel), no-block→tool
                if (state.blockStarted) {
                    events.push({ type: "content_block_stop", index: state.blockIndex });
                    state.blockIndex++;
                    state.blockStarted = false;
                }
                state.toolIndexMap.set(openaiIdx, state.blockIndex);
                state.blockStarted = true;
                state.currentBlockType = 'tool_use';
                events.push({
                    type: "content_block_start",
                    index: state.blockIndex,
                    content_block: {
                        type: "tool_use",
                        id: toolCall.id || `tool_${uuidv4()}`,
                        name: toolCall.function.name,
                        input: {}
                    }
                });
            }
            // Argument deltas route to the already-open block via toolIndexMap
            if (toolCall.function?.arguments) {
                const blockIdx = state.toolIndexMap.get(openaiIdx) ?? state.blockIndex;
                events.push({
                    type: "content_block_delta",
                    index: blockIdx,
                    delta: { type: "input_json_delta", partial_json: toolCall.function.arguments }
                });
            }
```

KEY BEHAVIORAL INVARIANT: The `toolCall.function?.arguments` block is unconditional —
it fires for ANY chunk (name present or absent). This is correct: argument deltas always
use the blockIdx from toolIndexMap, which was set when the block was first opened.

SCOPE: Touch ONLY the tool call section (~lines 581-615). Do not change the text block
handling (lines 619-631), the finish_reason block (lines 635-637), or any other code.

STATE STRUCTURE (for reference):

```javascript
    blockIndex: 0,
    toolIndexMap: new Map(),   // openaiIdx → blockIndex (set once per index)
    messageId: ...
}
```
