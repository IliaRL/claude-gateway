<objective>
Fix the empty-tool-name streaming bug in OpenAIConverter.toClaudeStreamChunk so that
content_block_start for a tool_use block is emitted exactly once per tool index.

Purpose: When an OpenAI provider streams tool calls, the first chunk carries a non-empty
function.name and subsequent chunks carry empty function.name with more argument JSON.
The current code at line 581 only emits content_block_start when toolCall.function?.name
is truthy — which is correct. However, it does NOT currently check whether
toolIndexMap.has(openaiIdx) before opening a new block. If a provider sends the name
in multiple chunks, or if the condition logic has an edge case, a duplicate block_start
can fire. The fix hardens the guard: content_block_start for tool_use is only emitted if
toolIndexMap does NOT already have openaiIdx.

Output: OpenAIConverter.js with the tool block guard hardened at lines ~581-606.
</objective>

CURRENT code around lines 578-616 (the tool call section inside toClaudeStreamChunk):
```javascript
            if (toolCall.function?.name) {
                // New tool call starting: close any open text/thinking block first.
                if (state.blockStarted && state.currentBlockType !== 'tool_use') {
                    events.push({ type: "content_block_stop", index: state.blockIndex });
                    state.blockIndex++;
                    state.blockStarted = false;
                } else if (state.blockStarted && state.toolIndexMap.has(openaiIdx)) {
                    // Parallel tool: close previous tool block before opening next.
                    events.push({ type: "content_block_stop", index: state.blockIndex });
                    state.blockIndex++;
                    state.blockStarted = false;
                }
                state.toolIndexMap.set(openaiIdx, state.blockIndex);
                state.blockStarted = true;
                state.currentBlockType = 'tool_use';
                events.push({
                    type: "content_block_start",
                    index: state.blockIndex,
                    content_block: {
                        type: "tool_use",
                        id: toolCall.id || `tool_${uuidv4()}`,
                        name: toolCall.function.name,
                        input: {}
                    }
                });
            }

            if (toolCall.function?.arguments) {
                const blockIdx = state.toolIndexMap.get(openaiIdx) ?? state.blockIndex;
                events.push({
                    type: "content_block_delta",
                    index: blockIdx,
                    delta: { type: "input_json_delta", partial_json: toolCall.function.arguments }
                });
            }
```

THE BUG: The outer `if (toolCall.function?.name)` guard prevents duplicate block_start
for same-chunk scenarios. BUT: if a provider sends the same tool index across multiple
chunks each with a non-empty name (some providers repeat the name on every argument delta),
the guard fires multiple times. The inner `else if (state.blockStarted && state.toolIndexMap.has(openaiIdx))`
closes and reopens — creating a duplicate block_start with the same name.

THE FIX: Add `!state.toolIndexMap.has(openaiIdx)` to the outer condition. A new tool block
should only be opened if this openaiIdx has NOT been seen before:

```javascript
            if (toolCall.function?.name && !state.toolIndexMap.has(openaiIdx)) {
                // New tool call starting: close any open text/thinking block first.
                if (state.blockStarted && state.currentBlockType !== 'tool_use') {
                    events.push({ type: "content_block_stop", index: state.blockIndex });
                    state.blockIndex++;
                    state.blockStarted = false;
                }
                // Note: removed the 'else if toolIndexMap.has(openaiIdx)' branch — it is now
                // unreachable because the outer guard already blocks on has(openaiIdx).
                state.toolIndexMap.set(openaiIdx, state.blockIndex);
                state.blockStarted = true;
                state.currentBlockType = 'tool_use';
                events.push({
                    type: "content_block_start",
                    index: state.blockIndex,
                    content_block: {
                        type: "tool_use",
                        id: toolCall.id || `tool_${uuidv4()}`,
                        name: toolCall.function.name,
                        input: {}
                    }
                });
            }
            // Argument deltas route to the already-open block via toolIndexMap
            if (toolCall.function?.arguments) {
                const blockIdx = state.toolIndexMap.get(openaiIdx) ?? state.blockIndex;
                events.push({
                    type: "content_block_delta",
                    index: blockIdx,
                    delta: { type: "input_json_delta", partial_json: toolCall.function.arguments }
                });
            }
```

KEY BEHAVIORAL INVARIANT: The `toolCall.function?.arguments` block is unconditional —
it fires for ANY chunk (name present or absent). This is correct: argument deltas always
use the blockIdx from toolIndexMap, which was set when the block was first opened.

SCOPE: Touch ONLY the tool call section (~lines 581-615). Do not change the text block
handling (lines 619-631), the finish_reason block (lines 635-637), or any other code.
