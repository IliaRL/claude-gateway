# Plan 01-02: Add Smoke Script + Connectivity One-Liners

## Objective

Add `pnpm run smoke` as the primary one-command proxy validation entry point.
Add `pnpm run check:models` and `pnpm run check:chat` as the two curl-based
connectivity one-liners. Document that `pnpm run verify:quick` is the 30s gate.

## Context

- `live-verify.cjs --quick` already exists and is accurate, quota-safe, and semantically validates responses
- `verify:live` (full suite) and `verify:quick` (one per provider, chat only) already exist
- Missing: `smoke` alias, and discoverable curl one-liners for manual spot-checks
- The curl one-liners should be npm scripts so `pnpm run` lists them — easy to find and copy

## Tasks

<task id="01-02-T1" type="execute">
<title>Add smoke, check:models, and check:chat npm scripts to package.json</title>
<read_first>
- AIClient2API/package.json (full file — read before editing to match existing style and avoid duplicating existing scripts)
- AIClient2API/scripts/live-verify.cjs (lines 1-50 — confirm --quick behavior before aliasing)
</read_first>
<action>
In AIClient2API/package.json, add these three entries to the "scripts" section,
after the existing "verify:quick" entry:

  "smoke": "node scripts/live-verify.cjs --quick",
  "check:models": "curl -sf -H \"Authorization: Bearer $AICLIENT_TOKEN\" http://127.0.0.1:3000/v1/models | jq '.data[].id' | head -10",
  "check:chat": "curl -sf http://127.0.0.1:3000/v1/messages -H \"Authorization: Bearer $AICLIENT_TOKEN\" -H \"Content-Type: application/json\" -d '{\"model\":\"claude-sonnet-4-6\",\"max_tokens\":16,\"messages\":[{\"role\":\"user\",\"content\":\"Hi\"}]}' | jq '.content[0].text'"

Do NOT add comments inside JSON (invalid). Keep the JSON strictly valid.
Do NOT change any existing scripts (verify:live, verify:quick, test, start, etc.).
</action>
<acceptance_criteria>
- `python3 -c "import json; d=json.load(open('AIClient2API/package.json')); print(d['scripts']['smoke'])"` prints `node scripts/live-verify.cjs --quick`
- `python3 -c "import json; d=json.load(open('AIClient2API/package.json')); print(d['scripts']['check:models'])"` prints the curl command for /v1/models
- `python3 -c "import json; d=json.load(open('AIClient2API/package.json')); print(d['scripts']['check:chat'])"` prints the curl command for /v1/messages
- `python3 -c "import json; json.load(open('AIClient2API/package.json'))"` exits 0 (valid JSON)
- All pre-existing scripts remain unchanged
</acceptance_criteria>
</task>

## Notes

- `check:models` and `check:chat` require `$AICLIENT_TOKEN` to be set in the shell — this is
  already exported in ~/dotfiles/zsh/zshrc as `AICLIENT_TOKEN`
- These scripts use `jq` for output — jq must be installed (`which jq` confirms this)
- The check:chat script uses claude-sonnet-4-6 as a fixed model ID — if this model is
  unavailable, the proxy will fall back per modelFallbackMapping in config.json
